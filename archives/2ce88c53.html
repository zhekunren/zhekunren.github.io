1 引言

前几天，在”Mysql安装与基础使用（QT、Matlab）”中备分了Mysql和Qt的配置，觉得还是需要写一个demo-project。

1.1 概述

本项目设计并实现了一个精简的多用户图书管理系统，应用后端通过与 mysql 进行连接实现对数据的管理，数据存储在本地。游客可以查阅图书信息；用户可以登陆、注册、查看图书，借书、还书等操作；管理员可以通过登陆界面实现对用户和书籍的管理，具有添加/删除用户/书籍等权限。

1.2 仓库地址

library-manage-by-mysql-qt

2 项目介绍

2.1 整体功能描述

游客： 只能查询和检索相应的书籍信息，不能借阅书籍；
用户： 能够查询相应的书籍信息，需要在登陆后才能借阅、归还书籍；
管理员： 可以查询相应书籍信息，在登录后拥有对书籍和用户的管理权限。能够查询、插入、更新、删除书籍，既可以单本插入，也可以通过脚本文件批量入库。能够查询、插入、更新、删除用户。 查询用户借书情况等。

2.2 功能详细介绍

2.2.1 游客

标题栏显示“游客”。
1.图书搜索页面
当用户为游客时，只具有查询书籍的权限。所以只有一个“图书搜索”页面，且表格下方不会出现“借阅书籍”按钮，此时只能进行检索书籍，可以看到书籍的信息以及库存/借阅数量。
书籍检索采用条件检索的方式，默认为检索“所有类型”的书籍。检索条件为左侧种类和右侧输入的条件“相与”,点击“开始检索”按钮，即可在列表中会列出数据库中所有书籍的信息。点击“恢复默认”，即将左侧种类条件恢复为“所有类型”，而右侧检索条件清空。

2.用户注册界面
当用户为游客时，可以点击标题栏的“注册”按钮，进行用户注册。

3.用户登录界面
当用户为游客时，可以点击标题栏的“登录”按钮，进行用户/管理员登录。

2.2.2 用户

标题栏显示用户名。
1.图书搜索页面
当用户为用户时，“图书搜索”页面中表格下方新出现“借阅书籍”按钮。用户可以通过勾选表中的Checkbox来选择要借阅的书籍。

2.借阅情况页面
当用户为用户时，会出现“借阅情况”页面。页面中的表格显示当前用户借阅书籍的情况，可以通过表中Checkbox以及表格下方的“全选”、“归还书籍”按钮来归还书籍。

2.2.3 管理员

1.用户管理页面
“用户管理页面”界面左侧为管理部分，右侧表格列出每个用户的基本情况。表格中的Checkbox配合左侧“更改用户权限”和“删除用户”按钮，可以进行对应的操作。当点击“添加新用户”按钮时，出现用户注册界面的界面。

2.用户详细信息界面
“用户管理页面”界面左侧，输入用户学号，点击“查询/更改”按钮后，弹出“用户详细信息界面”，展现了对应学号的用户的详细信息。页面中，除了借阅书籍的数量以及表中数据不能改变，其他数据均可以修改，点击“更改用户信息”按钮后，经过冗错检测后，即可完成修改。此外，当用户没有借阅书籍时，可以点击“删除用户”按钮来进行删除操作。

3.书籍管理页面
“书籍管理页面”页面左侧为管理部分，右侧表格列出书籍的基本情况。表格中的Checkbox配合左侧“删除书籍”按钮，可以进行删除书籍的操作。

4.书籍详细信息界面
“书籍管理页面”界面左侧，输入书籍编号，点击“查询/更改”按钮后，弹出“书籍详细信息界面”，展现了对应编号的书籍的详细信息。页面中，除了书籍已借出的数量以及表中数据不能改变，其他数据均可以修改，点击“更改书籍信息”按钮后，经过冗错检测后，即可完成修改。此外，当书籍没有被借阅时，可以点击“删除书籍”按钮来进行删除操作。

5.书籍添加/更新界面
“书籍管理页面”界面左侧，点击“添加书籍”按钮后，弹出“书籍添加/更新界面”。
界面的上侧信息填写完成后，用于添加/更新单册书籍，当书籍编号在数据库中不存在时为添加，否则存在时为更新，且更新时需要保证更新后的数量大于等于已借出的数量。
界面下侧为批量添加多本书籍，既可以直接在表格中填写，也可以利用txt文件脚本来批量添加。其中，文件脚本的格式为 不同书籍信息需要通过换行（即”\n”）隔开，每个书籍中数据的顺序按照表格顺序，且通过一个空格分割开，书籍数据也需要完整。此外，可以通过“清空”按钮来清除表格数据，通过“添加/更新书籍”按钮来更新多个书籍。

2.3 代码结构简要介绍

项目代码中，每个界面都是一个类，共有10个界面类。此外，将所使用到的数据封装在datastructure.h中。具体实现过程，请详细看代码。

指针的操作需要特别注意，因为常常在写程序时会容易出现内存泄漏或操作野指针的情况。
在本工程中，所有类的构造函数中会对类中指针进行初始化为NULL，在析构函数中会判断指针是否指向了开辟的内存，从而进行delete操作。在“MainWindow界面”中，除了分配类中变量的内存外，还会分配“window界面”和“widget页面”的内存空间。为了方便管理，对于“window界面”，通过关闭事件触发的函数“void closeEvent(QCloseEvent *event)”来向“MainWindow界面”发送信号，从而在相应槽函数中自动delete对应“window界面”内存，删除后设置为NULL，避免野指针；而对于“widget页面”，则是手动delete和赋值NULL，操作同类中其他指针一样；此外，在“MainWindow界面”的关闭事件触发函数“void closeEvent(QCloseEvent *event)”中会对类中的指针多指向的内存进行清理，防止内存泄漏。

2.4 补充QT打包生成exe的方法

首先需要两个工具。一个是QT5自带的windeployqt（这个大家不需要下载安装），另外一个是需要安装Engima Virtual Box，这是下载链接Engima Virtual Box。接下来介绍详细步骤：
1.Qt采用Release的方式生成一次，找到生成后的文件夹，我的在工程目录下LibraryManageByMysqlQt\release。打开后，复制release目录下的.exe程序到一个新建的文件夹，例如我的新建的文件夹为F:\03\NormalStudy\TechAcc\Mysql\library-manage-by-mysql-qt\EXE\Compass。
2.windows搜索中，搜索QT，找到Qt 5.5 for Desktop (MinGW 4.9.2 32 bit)，打开命令窗口。切换路径到前面复制有.exe程序的新建文件夹。接着，输入打包命令windeployqt 程序名。以我的为例，执行

待运行结束，点击新建目录下的exe程序就可以运行了。
3.打开运行Enigma Virtual Box，在Enter Input File Name处选择打包程序的文件，即前面新建目录中的.exe文件。接着，点击Add...按钮–>Add Folder Recursive，然后选择新建目录，即.exe所在目录。最后配置File Options中将Compress Files选中后，点击Process，等待结束，即可看到生成的打包好的文件LibraryManageByMysqlQt_boxed.exe。
