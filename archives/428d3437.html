搭建稳定快速、高效免费的图床

参考链接：GitHub + jsDelivr + PicGo + Imagine 打造稳定快速、高效免费图床

1.新建一个 GitHub 仓库，将权限设置成 public
2.生成并获取 Token。
① 在Github页面，点击用户头像，选择 Settings -> 点击 Developer settings -> 点击 Personal access tokens -> 点击 Generate new token。
② 填写 Token 描述（Note框，我的是blog_image（用于图库）），只勾选 repo，然后点击 Generate token 生成一个 Token。
③ 复制 Token 密钥，注意：这个 Token 只会显示一次。
3.配置 PicGo 并使用 jsdelivr 作为 CDN 加速。
前往下载 PicGo ，安装好后开始配置图床。
① 设定仓库名：按照 Github用户名 / 图床仓库名 的格式填写,我的是zhekunren/blog_image；
② 设定分支名：master ;
③ 设定 Token：粘贴之前生成的 Token
④ 指定存储路径：填写想要储存的路径，如 img/，这样就会在仓库下创建一个名为 img 的文件夹，图片将会储存在此文件夹中。注意：末尾要加/ ；
⑤ 设定自定义域名：作用是在图片上传后，PicGo 会按照自定义域名 + 上传的图片名的方式生成访问链接，放到粘贴板上，因为我们要使用 jsDelivr 加速访问，所以可以设置为 https://cdn.jsdelivr.net/gh/ 用户名 / 图床仓库名。我的填写是https://cdn.jsdelivr.net/gh/zhekunren/blog_image

jsDelivr 为开发者提供免费公共 CDN 加速服务

4.一般，图片大小都是超过 200KB 的，所以网页加载的时候会特别慢，所以上传前会对图片进行压缩。
① Imagine，需要注意的，Imagine 采用的是有损压缩
② 无损压缩网站，tinypng
